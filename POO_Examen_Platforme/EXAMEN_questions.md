# Feuille d'examen — Instructions pour l'étudiant

Vous devez rendre :
1. Un dossier zip nommé `POO_Examen_Platforme_<votre_nom>.zip` contenant le code source, README, et les diagrammes UML.
2. Un fichier `Rendu.md` expliquant vos choix et répondant aux questions ci-dessous.

## Questions à rendre (obligatoire)
1. **Diagramme de classes complet** (UML) — inclure : classes, attributs, méthodes publiques, relations d'héritage, interfaces, exceptions.
2. **Décrire le rôle de chaque méthode** — pour chaque méthode listée dans README, fournir préconditions/postconditions et exceptions possibles.
3. **Diagramme de cas d'utilisation** — acteurs (Auteur, Administrateur, Visiteur), cas (publier, cacher, télécharger, participer à un événement).
4. **Diagramme de séquence** pour le scénario "Ajout d'un contenu" (depuis la soumission par un auteur jusqu'à publication).
5. **Implémentation Java** — toutes les classes et exceptions (compilables).
6. **Gestion des exceptions** — démontrer au moins 3 scénarios où les exceptions sont levées et correctement gérées.
7. **Bonus (facultatif)** : interface CLI simple ou mini-UI; tests unitaires.

## Conseils
- Commentez votre code.
- Respectez le package `model` et `exceptions`.
- N'utilisez pas de frameworks externes.
- Livrez un README clair expliquant comment compiler et exécuter.

Bon travail !
